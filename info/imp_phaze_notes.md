# Задача для CODEX: добавить анализ производной фазы в измерение импеданса

## Контекст

В проекте реализуется измерение импеданса нагрузки через звуковую карту.

Схема измерения:

```text
OUT -> Rref -> точка измерения -> Load -> GND
```

Записываются два канала:

```text
CH1 = напряжение на выходе усилителя / перед Rref
CH2 = напряжение на нагрузке
```

Импеданс нагрузки считается комплексно:

```python
Zload = Rref * V2 / (V1 - V2)
```

Где `V1` и `V2` — комплексные спектры соответствующих каналов.

Сейчас строятся графики:

```text
|Zload|
phase current relative to voltage
```

Фаза тока относительно напряжения фактически равна:

```python
current_phase = -np.angle(Zload)
```

Нужно добавить дополнительный анализ: **производную фазы по логарифму частоты**. Наблюдение: в областях резонансов фаза быстро меняется, поэтому производная фазы должна хорошо подсвечивать резонансы, в том числе мелкие, которые не всегда очевидны на графике модуля импеданса.

## Что нужно сделать

Добавить вычисление:

```python
dphi_dlogf = d(phi) / d(log10(f))
```

Единицы измерения:

```text
deg/decade
```

То есть “градусов на декаду частоты”.

## Требования к реализации

### 1. Фаза должна быть unwrap

Перед дифференцированием фазу нужно разворачивать:

```python
phase_rad = np.unwrap(np.angle(Zload))
phase_deg = np.rad2deg(phase_rad)
```

Для фазы тока:

```python
current_phase_deg = -phase_deg
```

### 2. Производную считать по логарифму частоты

Не использовать обычное `dphi / df`, потому что частотная ось логарифмическая и низкочастотные резонансы будут непропорционально раздуты.

Правильно:

```python
x = np.log10(freq)
dphi_dlogf = np.gradient(current_phase_deg, x)
```

или для фазы импеданса:

```python
dphi_dlogf = np.gradient(phase_deg, x)
```

В графиках логичнее использовать фазу тока, потому что она уже отображается в текущем интерфейсе:

```python
d_current_phase_dlogf = np.gradient(current_phase_deg, np.log10(freq))
```

### 3. Сглаживание

Производная сильно усиливает шум, поэтому необходимо предусмотреть сглаживание.

Минимальный вариант:

```python
phase_for_derivative = smooth(current_phase_deg)
dphase = np.gradient(phase_for_derivative, np.log10(freq))
```

Если в проекте уже есть логарифмическое сглаживание, использовать его.

Важно: сглаживать фазу после `unwrap`, но до `gradient`.

Если готового удобного сглаживания для вещественного массива нет, добавить простую функцию, например:

```python
def smooth_real_logfreq(freq, y, sigma=1.0):
    ...
```

Но не усложнять. Для первого POC допустимо использовать существующий фильтр/сглаживание проекта.

### 4. Добавить график

В итоговый график добавить третью кривую или отдельную ось/подграфик:

```text
|Zload|
current phase, deg
d(current phase) / dlog10(f), deg/decade
```

Предпочтительно сделать отдельный subplot, чтобы график не превращался в лапшу:

```text
subplot 1: |Z|, Ohm
subplot 2: current phase, deg
subplot 3: d phase / d log10(f), deg/decade
```

Ось X везде логарифмическая.

Подпись третьего графика:

```text
d current phase / d log10(f), deg/decade
```

### 5. Добавить resonance score

Добавить расчет простой метрики:

```python
phase_derivative_score = np.abs(d_current_phase_dlogf)
```

Она пока нужна только для визуального анализа и будущего поиска пиков.

Опционально можно построить её вместо signed derivative или вместе с ней.

Для первого варианта лучше вывести signed derivative, потому что знак тоже может быть полезен.

### 6. Не ломать существующий пайплайн

Существующие графики и расчёт импеданса должны остаться рабочими.

Добавить флаг/параметр:

```python
plot_phase_derivative = True
```

или аналогичный, в зависимости от архитектуры текущего кода.

Если CLI нет, оставить просто константу рядом с настройками построения графиков.

### 7. Обработка краёв диапазона

Перед вычислением производной исключить нулевые и отрицательные частоты:

```python
mask = freq > 0
```

Также желательно исключить точки, где комплексный импеданс содержит `nan` или `inf`.

```python
mask = (freq > 0) & np.isfinite(Zload.real) & np.isfinite(Zload.imag)
```

### 8. Минимальная диагностика

После расчета вывести в консоль несколько крупнейших фазовых событий:

```text
Top phase derivative peaks:
freq = ... Hz, dphi/dlogf = ... deg/decade, |Z| = ... Ohm
```

Для поиска пиков использовать `scipy.signal.find_peaks`:

```python
from scipy.signal import find_peaks

score = np.abs(d_current_phase_dlogf)

peaks, props = find_peaks(
    score,
    prominence=np.percentile(score, 90) * 0.2,
    distance=5,
)
```

Выводить не больше 10 самых заметных пиков, отсортированных по `score`.

Если `find_peaks` уже не импортирован в проекте, добавить импорт.

### 9. Цель проверки

На измерениях динамика должны стать заметны:

* основной механический резонанс;
* дополнительные низкочастотные пики;
* мелкие фазовые события в области средних частот;
* переход к индуктивному характеру на ВЧ.

Особенно важно проверить второй график из текущих измерений, где есть несколько резонансов внизу: производная фазы должна сделать их более очевидными.
