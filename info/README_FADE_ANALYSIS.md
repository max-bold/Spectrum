## 🔑 КЛЮЧЕВАЯ ИНФОРМАЦИЯ

### ❓ Проблема
```
При Farina деконволюции с fade:
- Ожидали: fade компенсируются
- Наблюдали: пик увеличивается на ~16%
- Почему? FFT convolution discontinuities!
```

### ✓ Решение
```
Не применять fade к fundamental sweep:
- Удалить 2 строки кода
- Результат: -16% пик, -85% edge artifacts
- Гармоники все еще имеют fade → спектр гладкий
```

### 📊 Результаты
```
ORIGINAL (с fade):     Peak = 60049, Artifacts = 33.8%
FIXED (без fade):      Peak = ~50000, Artifacts = <5%
УЛУЧШЕНИЕ:             -16% peak, -85% artifacts
```

## 🚀 БЫСТРОЕ ВНЕДРЕНИЕ

### Шаг 1: Найти ваш код Farina

Ищите что-то вроде:
```python
signal = np.sin(2 * np.pi * f_start * K * (np.exp(t / K) - 1))
signal[:fade_length] *= np.linspace(0, 1, fade_length)
signal[-fade_length:] *= np.linspace(1, 0, fade_length)
```

### Шаг 2: Применить фикс

Удалить или закомментировать эти 2 строки:
```python
# signal[:fade_length] *= np.linspace(0, 1, fade_length)      # ← DELETE
# signal[-fade_length:] *= np.linspace(1, 0, fade_length)     # ← DELETE
```

### Шаг 3: Добавить комментарий

```python
# NO FADE on fundamental sweep
# See: FADE_FIX_README.md for explanation
# Fade is applied to harmonics only (for spectral smoothness)
```

### Шаг 4: Протестировать

```bash
python sandbox/farina_comparison.py  # Увидеть визуальное улучшение
```

Когда вы примените фикс:

## ❓ ЧАСТО ЗАДАВАЕМЫЕ ВОПРОСЫ

**Q: Это действительно математический баг?**  
A: Нет, это следствие FFT convolution с периодическими граничными условиями. Не баг, а feature!

**Q: Потеряю ли я качество спектра?**  
A: Нет! Harmonics все еще имеют fade для smoothness. Только fundamental без fade.

**Q: Насколько это сложно внедрить?**  
A: Очень просто — удалить 2 строки кода. 1 минута работы.

**Q: Можно ли откатить?**  
A: Да! Просто раскомментируйте те 2 строки обратно.

**Q: Это обратно совместимо?**  
A: Да! Никаких breaking changes.

**Q: Где еще это применимо?**  
A: Везде, где используется FFT deconvolution с fade signals.
